1.2 Python syntax and formatting Conventions A Python file being formatted improperly or having inappropriately named components will cause it to not function.

The most common errors you will face as you program will be syntax errors. Even if your logic is sound, if you spell something incorrectly your program will not function. Computers are good at following instructions but mistakes as minor as writing rint(“Hello”) instead of print(“Hello”) will trip up the computer.

Indentation indicates a line/block of code is related to the non-indented code directly above it. The following code will not work due to the unnecessary indent

print(“Hi.”)
print(“My name is John.”)

Starting in unit 3 cases where indentation is required will appear.

Ensure you include the correct spelling, spacing, indents, and within your code.
